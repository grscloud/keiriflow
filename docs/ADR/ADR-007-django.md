# ADR-007 Djangoを採用する

## 背景

Webアプリケーションを短期間で開発する必要がある。

---

## 決定

バックエンドフレームワークとしてDjangoを採用する。

---

## 採用理由

- Pythonエコシステムを活用できる
- Django Adminを利用できる
- ORMが成熟している
- 認証機能が標準搭載されている
- セキュリティ対策が充実している

---

## 採用しなかった案

FastAPI

理由

API性能は高いが、管理画面や認証機能を個別実装する必要があり、初期開発コストが増加する。

Laravel

理由

チーム内のPythonスキルを活用できるDjangoを優先した。

Spring Boot

理由

Javaは開発速度よりも大規模開発向けであるため、本プロジェクトでは採用しない。

---

## 移行条件

現時点では変更予定なし。
